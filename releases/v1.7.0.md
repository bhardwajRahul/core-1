## StaticBackend v1.7.0

This release continues the restart work from `v1.6.3`, with most of the effort going into account/user administration, server-side function capabilities, query improvements, and operational fixes found while running StaticBackend again.

## In this release

### Account and user management

StaticBackend now supports cross-account user associations with a role per account. This makes it possible for the same user to access multiple accounts without losing account-specific permissions.

There are also new administration paths for changing user roles, changing user email addresses across the associated account records, getting a user by id with root access, and getting auth tokens for account/user associations when promoting or managing users.

When a user logs in to an account where the account/user association is missing, StaticBackend now creates that association automatically.

### Server-side functions

Server-side functions can now react to account creation through a system event, and there is a root execution handler for functions that need to operate across all accounts.

The runtime also includes the missing bulk database helpers, including `updateBulk` and `deleteBulk`, making server-side data maintenance workflows more complete.

Full-text search support from functions was tightened up too, with more robust indexing and deletion behavior.

### Query and database improvements

Queries now support `contains` and `!contains`, plus filtering by another field value. The query parser and providers now carry more explicit type information, and database indexes can be created with the proper type so generated provider queries can use them correctly.

This release also includes storage usage and file listing per account, with provider implementations and tests across memory, MongoDB, PostgreSQL, and SQLite.

### Fixes and maintenance

There were a number of production and test fixes in this cycle:

* Pagination parameters are sanitized before use.
* Auth token expiration checks were fixed.
* Redis pub/sub handling was improved for dropped production connections.
* The memory database off-by-one behavior was fixed.
* MongoDB account user ObjectID handling was corrected.
* Email now uses quoted printable encoding for non ASCII characters.
* Upload size calculations now use 1024 factors.
* Test runner configuration was cleaned up for PostgreSQL and MongoDB runs.
* Go dependencies were refreshed, including `goja` and `golang.org/x/image`.

As usual, this release also includes linter fixes and test coverage for the touched providers and runtime behavior.
